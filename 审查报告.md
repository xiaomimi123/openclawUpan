## 一、高优先级（必须修复）

### 1. openclaw.zip 解压缺少路径遍历校验

**文件**: `main.js:164-184`
**风险**: 恶意 zip 可写入任意位置

微信插件解压有路径校验：
```javascript
if (!path.normalize(dest).startsWith(path.normalize(destDir))) {
  return resolve({ ok: false, error: '不合法的归档条目' })
}
```

但主 openclaw.zip 解压**没有这个校验**：
```javascript
const dest = path.join(openclawDir, entryName)  // ⚠️ 未校验
```

**修复**: 在 `fs.mkdirSync(dest)` 之前添加同样的路径校验逻辑。

---

### 2. 网关 Token 残留导致认证失败

**文件**: `main.js:430-434`
**风险**: 用户看到"已连接"但实际无法认证

场景：启动 → 崩溃 → 修复 → token 刷新失败 → `currentGatewayToken` 仍持有旧值 → UI 用旧 token 打开网页 → 认证失败。

```javascript
currentGatewayToken = await refreshGatewayToken()  // 失败返回 null
// 但 currentGatewayToken 可能还保留上次的旧值
```

**修复**: 启动前先清空 token：
```javascript
currentGatewayToken = null
const token = await refreshGatewayToken()
if (!token) return { ok: false, error: '...' }
currentGatewayToken = token
```

---

### 3. 安装前未检查磁盘空间

**文件**: `main.js:91-229`（`installToLocal` 函数）
**风险**: 解压到一半磁盘满，用户看到"安装失败"但不知道原因

preflight 检查在启动时做了磁盘空间检测（第 952 行），但 `installToLocal()` 没有。首次安装需要约 1.5 GB，如果 C 盘满了会静默失败。

**修复**: 在解压前检查目标盘剩余空间，不足时给出明确提示。

---

### 4. 静默 catch 导致问题无法诊断

**文件**: main.js 多处
**风险**: 用户报问题时无法定位原因

| 位置 | 被吞掉的错误 |
|------|------|
| 第 104 行 | `fs.rmSync` 清理旧安装失败（杀毒软件锁定） |
| 第 195 行 | Node.js runtime 复制失败（权限不足） |
| 第 228 行 | 版本标记写入失败（下次重复安装） |

所有 `catch {}` 应至少记录日志：
```javascript
catch (e) { console.warn('[installToLocal] cleanup failed:', e.message) }
```

## 二、中优先级（建议修复）

### 5. 表单验证过于宽松

**文件**: `setup.html`

| 字段 | 当前校验 | 问题 | 建议 |
|------|---------|------|------|
| API Key | 长度 > 8 | 太短，允许明显错误输入 | 按服务商设最低长度（anthropic >= 30） |
| Telegram Token | 长度 > 10 | 不检查格式 | 校验 `^\d{6,}:[A-Za-z0-9_-]+$` |
| 飞书 App ID | 长度 > 3 | 太短 | 校验 `^cli_[a-zA-Z0-9]+$` |
| Base URL | 长度 > 8 | 不校验 URL 格式 | 用 `new URL()` 验证 |
| 自定义 Model ID | 非空 | 允许纯空格 | 校验 `^[a-zA-Z0-9._-]+$` |

### 6. 定时器竞态条件

**文件**: `launcher.html` 多个 watchdog

启动时有两个独立定时器（30 秒 readyWatchdog + 60 秒 globalTimeout）。用户点"停止"后 `clearReadyWatchdog()` 只清了前者，60 秒定时器可能在停止后触发 `onGatewayReady()`，导致状态显示不一致。

**修复**: 统一清理所有 watchdog 定时器。

### 7. ANSI 正则每次调用重新编译

**文件**: `launcher.html` 微信登录/更新的 `stripAnsi()` 函数

正则定义在函数内部，每次收到输出都重新编译。高频输出时造成不必要的 GC 压力。

**修复**: 将正则提到函数外部作为常量。

### 8. refreshGatewayToken 错误信息丢失

**文件**: `src/config.js:213-233`

JSON 解析失败、文件写入失败、权限不足——都返回 `null`。主进程无法区分"配置损坏"和"磁盘写满"。

**建议**: 返回 `{ ok: false, error: '具体原因' }` 或抛出异常让调用者处理。

## 三、低优先级（可选改进）

### 9. 模型 ID 硬编码

`config.js` 中 `claude-sonnet-4-6`、`gpt-4o`、`deepseek-chat` 等写死。模型更新后需要改代码。

**建议**: 将默认模型存入 setup.json，允许未来通过 UI 选择。

### 10. 未签名 EXE 触发 SmartScreen

Windows 未签名应用首次运行会弹 SmartScreen 警告，小白用户可能被吓到。

**建议**: 购买代码签名证书，在 package.json 的 build 配置中添加签名。

### 11. API Key 摘要显示过多

**文件**: `setup.html:585`

确认页显示 API Key 前 6 位 + 后 4 位。短 Key 可能暴露过多信息。

**建议**: 只显示前 3 位 + 后 3 位，中间用固定长度掩码。

### 12. 缺少卸载方案

用户不知道 openclaw 装在 `%LOCALAPPDATA%\OpenClaw\`。如果想清除，没有卸载工具。

**建议**: 在启动器中加"卸载本机数据"功能，或提供独立卸载脚本。
